## ü§ñ ATS Compliance

All templates follow the latest 2025 ATS best practices from leading industry sources:

### What Makes These Templates ATS-Friendly?

1. **Single-Column Format**
   - ATS systems read left-to-right, top-to-bottom
   - Multi-column layouts often get scrambled by parsers
   - Our single-column design ensures proper data extraction

2. **Standard Section Headings**
   - "Professional Experience" not "Where I've Worked"
   - "Education" not "Academic Background"
   - ATS relies on these labels to categorize information

3. **Semantic HTML Structure**
   ```html
   <header> ‚Üí Name and contact
   <section> ‚Üí Each major resume section
   <h1> ‚Üí Your name
   <h2> ‚Üí Section titles
   <h3> ‚Üí Job titles
   <article> ‚Üí Individual experiences
   ```

4. **Font Selection**
   - **Body Text:** 10-12pt
   - **Headings:** 14-16pt
   - **Fonts:** Arial, Calibri, Georgia (universally recognized)

5. **No ATS-Breaking Elements**
   - ‚ùå No headers/footers for critical info
   - ‚ùå No text boxes or columns
   - ‚ùå No images, charts, or graphics
   - ‚ùå No tables for layout
   - ‚ùå No special symbols or icons

6. **Keyword Optimization**
   - Industry-specific terms throughout
   - Skills mentioned 2-3 times naturally
   - Action verbs for accomplishments
   - Quantifiable metrics included

## üé® Customization Guide

### Changing Colors

Each template uses a primary color for section headers and borders. Find and replace the hex code:

```css
/* Example: Marketing template uses #0984e3 */
color: #0984e3;  /* Change to your preferred color */
border-bottom: 2px solid #0984e3;  /* Also change border color */
```

**Professional Color Suggestions:**
- **Corporate Blue:** `#2c3e50` or `#3498db`
- **Professional Green:** `#27ae60` or `#16a085`
- **Executive Navy:** `#34495e` or `#2c3e50`
- **Tech Purple:** `#8e44ad` or `#6c5ce7`
- **Creative Orange:** `#e67e22` or `#f39c12`

### Adjusting Font Sizes

```css
/* Current settings */
h1 { font-size: 32px; }  /* Your name */
h2 { font-size: 14px; }  /* Section headers */
h3 { font-size: 12px; }  /* Job titles */
body { font-size: 11.5px; }  /* Body text */

/* Adjust if needed, but stay within ATS-safe ranges:
   Body: 10-12pt
   Headers: 14-16pt
   Name: 28-36pt */
```

### Adding/Removing Sections

1. Find the section to modify:
```html
<section>
    <h2>SECTION NAME</h2>
    <div class="section-content">
        <!-- Content here -->
    </div>
</section>
```

2. Copy entire `<section>` block to duplicate
3. Delete entire `<section>` block to remove
4. Maintain consistent spacing with other sections

### Modifying Content Length

**If content is too long:**
- Prioritize most recent/relevant experience
- Remove older positions (keep within 10-15 years)
- Consolidate bullet points
- Target 1-2 pages maximum

**If content is too short:**
- Expand bullet points with metrics
- Add relevant projects or certifications
- Include volunteer work or leadership roles
- Add professional affiliations

## üìö Best Practices

### Content Guidelines

1. **Use Action Verbs**
   - ‚úÖ "Increased revenue by 145%"
   - ‚ùå "Was responsible for revenue"

   Examples: Achieved, Built, Created, Developed, Engineered, Implemented, Led, Managed, Optimized, Spearheaded

2. **Quantify Everything**
   - Include numbers, percentages, dollar amounts
   - Before/after metrics show impact
   - User counts, team sizes, budget amounts
   - Time savings, efficiency gains

3. **Keyword Optimization**
   - Mirror job description language
   - Include industry-standard terms
   - Mention key skills 2-3 times naturally
   - Use both acronyms and full terms (e.g., "SEO" and "Search Engine Optimization")

4. **Tailor for Each Application**
   - Adjust summary to match job requirements
   - Reorder skills based on job posting
   - Highlight most relevant experiences
   - Remove irrelevant information

### Formatting Guidelines

1. **Consistency is Key**
   - Same date format throughout (e.g., "January 2020 - Present")
   - Consistent bullet point style
   - Uniform spacing between sections
   - Matching font weights and sizes

2. **White Space Matters**
   - Don't overcrowd the page
   - Leave breathing room around sections
   - Use line breaks strategically
   - Aim for 60-70% text coverage

3. **Bullet Points**
   - 3-6 bullets per position
   - Each bullet should be 1-2 lines
   - Most important accomplishments first
   - Start each with strong action verb

### File Export Tips

**For PDF Export:**
1. Use Chrome or Firefox (best rendering)
2. Print settings:
   - Paper: Letter (8.5" x 11")
   - Margins: Default
   - Scale: 100%
   - Background graphics: ON
3. File name: `FirstName_LastName_Resume.pdf`

**For Word (.docx) Conversion:**
1. Export to PDF first
2. Use online converter (Smallpdf, Adobe, etc.)
3. Verify formatting after conversion
4. Some ATS prefer .docx over PDF

## ‚ùì FAQ

### General Questions

**Q: Which template should I use?**
A: Choose based on your industry and career level. See the [Template Collection](#template-collection) table for guidance.

**Q: Can I use these for any country?**
A: These are optimized for US/Canada job markets. For international applications, you may need to adjust format (e.g., CV vs Resume, date formats, etc.).

**Q: Are these really ATS-friendly?**
A: Yes! They follow all 2025 ATS best practices from Jobscan, Resume.io, and other leading resume experts. Single-column layout, standard fonts, semantic HTML, and keyword optimization.

**Q: How do I know if my resume will pass ATS?**
A: Use free ATS scanners like Jobscan.co or Resume Worded to test your resume against specific job descriptions.

### Technical Questions

**Q: Do I need to know HTML to use these?**
A: No! You can edit them in any text editor. Just replace the sample text with your information while keeping the HTML tags intact.

**Q: Can I convert these to Word format?**
A: Yes, export to PDF first, then use an online PDF-to-Word converter. However, PDF is generally preferred for applications.

**Q: Will the colors affect ATS parsing?**
A: No! The colors are for visual presentation only. ATS systems parse the text content, which is all in standard HTML.

**Q: How do I add a photo?**
A: We don't recommend adding photos for US applications as they can cause ATS issues and potential bias. If required for international applications, consult an HTML tutorial for image embedding.

### Customization Questions

**Q: Can I change the layout to two columns?**
A: Not recommended! Two-column layouts confuse ATS systems. Stick with single-column for best results.

**Q: Can I add tables or charts?**
A: Avoid tables and charts as they're not ATS-friendly. Use bullet points and text to convey information.

**Q: How do I fit everything on one page?**
A: Reduce font size slightly (min 10pt), tighten spacing, or remove less relevant content. For experienced professionals, 2 pages is acceptable.

**Q: Can I use different fonts?**
A: Stick with Arial, Calibri, or Georgia for ATS compatibility. Other fonts may not parse correctly.
